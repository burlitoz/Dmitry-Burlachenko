## 📚 Любимая литература

Книги, которые сформировали мое понимание DevOps, Linux и инфраструктуры:

| Книга | Автор | О чем |
|-------|-------|-------|
| 🐧 **Внутреннее устройство Linux** | Дмитрий Кетов | Глубокое погружение в архитектуру Linux: ядро, процессы, память, файловые системы, управление ресурсами. Помогает понимать, как система работает "под капотом", а не просто запоминать команды. |
| 🛠 **Linux в действии** | Дэвид Клинтон | Практическое руководство по администрированию. Реальные сценарии: настройка серверов, управление процессами, работа с сетью, автоматизация на bash. Каждая глава — готовый инструмент для решения конкретной задачи. |
| 🚀 **Наука DevOps. Ускоряйся** | Джин Ким, Джез Хамбл, Патрик Дебуа | Манифест современного DevOps. Книга о том, как культура, автоматизация, измерение и шеринг (CALMS) меняют подход к разработке и эксплуатации. Вдохновляет строить системы, в которых разработчики и ops работают как единая команда. |
